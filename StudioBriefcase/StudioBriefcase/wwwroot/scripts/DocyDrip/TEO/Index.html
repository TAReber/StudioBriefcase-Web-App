DocyDrip - Text Editor Object
Rich Text Editor made with JavaScript

Web Text Editing solution Created by Tyler A Reber

  • Free and Open Source
  • Fully Customizable
  • Functionality enabled with Bitwise operations
  • Basic Editing Experience
  • Made with in-house traversal methods, without deprecated functions like the execCommand

Bit Values to Enable different interfaces

Value 0: Loads - No UI Enabledd
Value 1: DB Storage / Use Whitespace Formatting (Not Implemented)
 Value 2: Loads Bold, Italic, Underline
Value 4: Loads Strikethrough, Subscript and Superscripts
Value 8: Create Hyperlinks
 Value 16: Studio Briefcase Loads Color Selector
Value 32: Loads Background Color Selector
Value 64: Loads Font Size Selector
Value 128: Loads Font Face Selector
Value 256: Loads Loads Header Block Selector
Value 512: Loads Indents and Justifies
Value 1024: Loads List Buttons
Value 2047: Loads Everything
 Value 2048: No UI Enabled

Other Bits Reserved for additional Features.


Testing Editor on your Environment

Version 0.6 divided the text editor up into multiple JavaScript files. The goal was to separate the UI creation methods into a separate file which will allow people to easily customize the UI without needing to touch the Editor class.

Having multiple JavaScript export to one another means that the editor can only be tested on a running server or you will get a CORS policy error.

What is the CORS Error?

I understand the CORS policy as a layer of security that was created to prevent other people's JavaScript files from running code on people's personal computer all willynilly.

Solution

Solution is launch a server testing environment. I'm using this apache software to run a server for my testing environment. 

https://www.apachefriends.org/
